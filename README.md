# AI Recon Framework

Authorised-engagement, recon-only multi-agent system. Coordinates a team of
specialist AI agents and battle-tested security tools (nmap, subfinder, nuclei,
httpx, etc.) to map an organisation's external and internal attack surface.
Each tool call is scope-gated, logged to an append-only audit trail, and
executable only against targets inside the explicit scope you define.

## What it does

- **Passive + active recon only** — no exploitation, no payloads, no
  persistence, no lateral movement.
- Multi-agent team: Orchestrator (o3) plans; DNS, Network, Port, Fingerprint,
  Web, and OSINT agents run in parallel; a ValidationAgent filters noise before
  the orchestrator acts; a ReportAgent synthesises findings into a Markdown
  report.
- Findings are deduplicated and stored in a SQLite database per campaign run.
- Every tool execution is proxied through a Go sidecar (`recon-runner`) that
  enforces scope as an authoritative second gate and kills child processes
  immediately on campaign stop.
- Kill switch: `Ctrl-C` or `touch /tmp/.redteam_stop` stops all tool
  processes within one tool-call latency.

## Architecture

```text
cli.py
  └── OrchestratorAgent (o3)
        ├── DNSAgent           (Gemini 2.5 Flash)
        ├── NetworkMapperAgent (Gemini 2.5 Flash)
        ├── PortScannerAgent   (Gemini 2.5 Flash)   ← spawned per host
        ├── ServiceFingerprintAgent (GPT-4o-mini)   ← spawned per HTTP service
        ├── WebReconAgent      (Claude Sonnet 4.6)  ← spawned per web app
        ├── OSINTAgent         (Perplexity Sonar Pro)
        └── ReportAgent        (Gemini 2.5 Pro)

ValidationAgent (GPT-4o-mini) — critic that confirms noisy HOST/PORT/SUBDOMAIN
findings before the orchestrator reacts to them.

FindingsStore (SQLite) — shared blackboard with pub/sub, deduplication, and
two-tier subscription (raw → validated, rich findings auto-confirmed).

recon-runner (Go HTTP sidecar) — executes tool binaries with:
  - Scope enforcement (CIDRs + domain allowlist)
  - Per-tool goroutine semaphores (e.g. nmap ≤ 3 concurrent)
  - exec.CommandContext — child processes killed on context cancel
```

## Prerequisites

**API keys** (see `.env.example`):

| Variable | Purpose |
| --- | --- |
| `ANTHROPIC_API_KEY` | WebReconAgent (Claude) |
| `OPENAI_API_KEY` | OrchestratorAgent (o3), ServiceFingerprinter, ValidationAgent |
| `GOOGLE_API_KEY` | DNS, Network, Port, Report agents (Gemini) |
| `PERPLEXITY_API_KEY` | OSINTAgent |
| `SHODAN_API_KEY` | Optional — Shodan lookups via `shodan` CLI |

**Security tools** (required — will be checked at startup):

```text
nmap  subfinder  httpx  nuclei
```

**Recommended** (gracefully skipped if missing):

```text
amass  dnsx  masscan  rustscan  katana  ffuf  gowitness  gau  theHarvester  fierce
```

**Go 1.22+** — needed to build `recon-runner`.

## Setup

```bash
# 1. Copy and fill in API keys
cp .env.example .env
# edit .env

# 2. Install Python dependencies
pip install -r requirements.txt

# 3. Build the Go tool runner
make runner          # outputs ./recon-runner

# (Optional) install to PATH
make install
```

## Usage

```bash
# Map a network range
python cli.py --networks 10.0.0.0/24 --domains corp.example.com

# Multiple ranges + custom goal
python cli.py \
  --networks 10.0.0.0/24 192.168.1.0/24 \
  --domains corp.example.com staging.example.com \
  --exclude 10.0.0.1/32 \
  --goal "Focus on web-facing services and DNS footprint"

# Verbose (DEBUG) logging
python cli.py --networks 10.0.0.0/24 --verbose
```

**Kill at any time:**

```bash
Ctrl-C
# or from another shell:
touch /tmp/.redteam_stop
```

## Output

Each campaign writes to `runs/campaign_<timestamp>/`:

| File | Contents |
| --- | --- |
| `findings.db` | SQLite — all deduplicated findings |
| `audit.jsonl` | Append-only log of every decision and tool call |
| `report.md` | Markdown summary generated by ReportAgent |
